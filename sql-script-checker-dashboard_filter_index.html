檢查規則

目前 SQL 上線腳本檢查器套用的規則如下。

  1. NOLOCK 檢查：逐筆偵測每一段 SELECT 語句是否包含 NOLOCK，缺少時列入結果清單，並可定位高亮原始 SQL 片段。
  2. SELECT / UPDATE / DELETE + WHERE 檢查：逐筆偵測每一段 SELECT、UPDATE 或 DELETE 語句是否包含最外層 WHERE 條件。
  3. 註解排除：檢查時會先排除單行註解 -- 與區塊註解 /* ... */，避免註解文字影響判斷。
  4. 語句切分：支援以分號 ;、單獨一行 GO，以及最外層 SELECT、UPDATE、DELETE 關鍵字切分 SQL 語句。
  5. 略過項目：註解內容不檢查；暫存表 #table 不檢查；外層 SELECT ... FROM (...) 這類子查詢包裝不列為缺失。
  6. 檔案檢查：文件上傳支援 .sql、.txt、.log，可一次累積多份檔案後再檢查。
  7. 定位預覽：點擊缺失項目後，會在原文定位預覽中高亮對應 SQL 片段，並同步顯示行號。
